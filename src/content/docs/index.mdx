## The Story of This Course

You are a developer. You have built a microservice application — a React frontend, a Node.js API, a Redis cache, and a PostgreSQL database. It works perfectly on your laptop.

Now you need to deploy it. Not to one server — to a cluster of machines that can handle real traffic, recover from crashes, and scale when demand spikes.

**This course teaches you how.** Not by listing commands to memorize, but by walking you through the decisions, trade-offs, and reasoning behind every step.

<Aside type="tip" title="How This Course Works">
Every topic starts with **WHY** — the problem you are facing — before showing you HOW to solve it. You will never run a command without understanding why it exists.
</Aside>

## Who This Course Is For

This training is designed for **developers** who have:

- **Linux command-line experience** — you can navigate directories, edit files, and run commands
- **Software development experience** — in any language (JavaScript, Python, Go, Java, etc.)
- **Basic understanding of microservices** — you know what a frontend, backend, and database are

You do **not** need prior Docker or Kubernetes experience. We start from zero.

## Prerequisites

Before Day 1, make sure you have:

- A machine with at least **8 GB RAM** and a modern CPU
- **Docker** installed ([installation guide](/day1/))
- A **terminal** you are comfortable with
- A **text editor** or IDE
