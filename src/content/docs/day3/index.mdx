## The Story So Far

Over the last two days, you containerized our microservice application. You can run the API, Redis, and PostgreSQL together with Docker on a single machine.

But a single machine is a **single point of failure**. If it crashes, everything goes down. If traffic grows, you cannot add more capacity. You need a system that:

- Runs containers **across multiple machines**
- **Automatically restarts** crashed containers
- **Distributes traffic** across healthy instances
- Lets you **scale up and down** based on demand

That system is **Kubernetes**.
